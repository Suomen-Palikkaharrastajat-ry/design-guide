# Agent Guidance — Suomen Palikkaharrastajat ry Design Guide

This file is the primary entry point for all AI agents working in this repository.
Read it fully before making any changes.

---

## 1. Source of truth & precedence

When sources conflict, resolve in this order (highest to lowest):

1. **Tests** — if a test asserts behaviour, the behaviour is correct
2. **Source code comments** — local rationale
3. **This file** — general guidance

---

## 2. Agent self-guidance work loop

Before making any change:

1. Identify which source files are affected (see repository map below)
2. Run `cabal build` to confirm the Haskell project compiles before you start
3. Make the minimal change that satisfies the story / task
4. Add or update tests in `tests/` for any changed behaviour
5. Run `cabal test` and `hlint src tests` — both must pass
6. If Elm files changed, verify elm-pages compiles: `make assets && npx elm-pages build`

## 4. Change rules

- **No behaviour change without a test.** Add a test before or alongside any code change.
- **`Brand.Colors.hs` is the single source of truth** for colors. Never hard-code a hex value in another Haskell module; import from `Brand.Colors`. Mirror changes in `src/Brand/Colors.elm`.
- **`src/Brand/Generated.elm` is generated** by `cabal run logo-gen`. Never edit it by hand.
- **External tools** (`rsvg-convert`, `cwebp`, `img2webp`, `icotool`) are called via `System.Process`. Never add new binary dependencies without updating `devenv.nix`.
- **Do not commit generated files**: `design/`, `logo/`, `favicon/`, `brand.json`, `src/Brand/Generated.elm`, `dist/`, copied assets in `public/`.
- **Makefile targets**: use `make run` for the Haskell pipeline, `make site` for the full build, `make dev` for local development. Run `make install` once after checkout.
- **`TODO.md` is gitignored** (local working notes only). Committed reference: `TODO.md.example`.

---

## 5. Decision escalation rules

Escalate (ask the user) when:

- A change would modify the `brand.json` schema (field names, nesting, array order)
- A change would alter the set of output files or their naming convention
- A change requires a new external binary dependency
- A change removes or renames a public function in any `Brand.*` or `Logo.*` module
- Two ADRs conflict and there is no clear precedence

## 6. Design token machine-readability

This project provides two complementary machine-readable artefacts:

| File | Format | When to use |
|------|--------|-------------|
| `design-guide.json` | W3C Design Tokens (monolithic) | Full spec in one fetch; legacy |
| `design-guide/index.jsonld` | JSON-LD (split) | Start here; follow `sections[@id]` to fetch only what you need |
| `design-guide/colors.jsonld` | JSON-LD ColorToken | Colour decisions |
| `design-guide/typography.jsonld` | JSON-LD TypographyStyle | Font/size decisions |
| `design-guide/spacing.jsonld` | JSON-LD SpacingToken | Layout decisions |
| `design-guide/motion.jsonld` | JSON-LD MotionToken | Animation decisions |
| `design-guide/logos.jsonld` | JSON-LD LogoVariant | Logo file selection |
| `design-guide/components.jsonld` | JSON-LD ComponentSpec | UI code generation |
| `design-guide/context.jsonld` | JSON-LD @context | Vocabulary; resolve before parsing tokens |

**JSON-LD usage pattern:**
1. Fetch `design-guide/index.jsonld`
2. Resolve `@context` from `context.jsonld`
3. Follow `sections` links to fetch only the sections you need
4. Each token has `@id` (stable URI), `@type`, `name`, `value`, `description`

`design-guide.json` conforms to the [W3C Design Tokens Community Group](https://tr.designtokens.org/) conventions; JSON-LD section files follow the same vocabulary mapped to Linked Data.

### 6.1 Token object format

Individual tokens use three standard fields:

| Field | Required | Meaning |
|-------|----------|---------|
| `$value` | yes | The token's resolved value (hex string, px int, ms int, etc.) |
| `$type` | recommended | W3C type hint: `color`, `dimension`, `duration`, `cubicBezier`, `typography`, `number` |
| `$description` | recommended | Full sentence: what the token is for and when to use it vs. alternatives |

Non-token grouping objects (arrays, shape objects) do **not** use `$`-prefixed keys.

### 6.2 Color tokens (`colors.*`)

| Path | Content |
|------|---------|
| `colors.brand` | Primitive brand colors (LEGO Black, LEGO White). Each entry has `hex`, `id`, `name`, `$description`, `usage[]`, and `wcag` sub-object. |
| `colors.skinTones` | Skin-tone palette. Each entry has `wcag.onWhite` and `wcag.onBlack` with numeric ratios and WCAG rating strings. |
| `colors.rainbow` | Decorative-only colors (`decorativeOnly: true`). Do not use as text color. |
| `colors.semantic` | **Use this layer in UI code.** Maps intent to hex: `text.primary`, `text.onDark`, `text.muted`, `text.subtle`, `background.page`, `background.dark`, `background.subtle`, `background.accent`, `border.default`, `border.brand`. Each entry has `$value`, `$type: "color"`, optional `$ref` to the primitive it aliases, and `$description`. |

Rules:
- **Never hard-code a hex value** in Elm, CSS, or Haskell. Reference `Brand.Colors` (Haskell), `Brand.Generated` or `Brand.Colors` (Elm), or CSS custom properties (`--color-brand`, `--color-brand-yellow`).
- When choosing a color, prefer a `colors.semantic` token over a `colors.brand` primitive.
- Check `wcag.onWhiteRating` before placing text: anything rated `"fail"` must not be used as foreground text on a white or light background.

### 6.3 Typography tokens (`typography.*`)

| Path | Content |
|------|---------|
| `typography.primaryFont` | Font family, variable axes (wght 100–900), file path, license (OFL-1.1). |
| `typography.scale` | Array of named type styles. Each entry: `name`, `$type: "typography"`, `fontWeight`, `fontSizeRem`, `fontSizePx`, `lineHeight`, `letterSpacingEm`, `cssClass`, `$description`. |
| `typography.usageRules` | Prose rules as a string array. |

Named type styles: `Display`, `Heading1`, `Heading2`, `Heading3`, `Body`, `BodySmall`, `Caption`, `Mono`, `Overline`.

Rules:
- Use `typography.scale[*].cssClass` as the Tailwind class string for that style.
- Minimum body text: `fontSizePx >= 16`. Minimum label/caption: `fontSizePx >= 14`.
- Font is self-hosted at `/fonts/Outfit-VariableFont_wght.ttf`; do not reference external CDNs.

### 6.4 Spacing tokens (`spacing.*`)

| Path | Content |
|------|---------|
| `spacing.baseUnit.$value` | `4` (px). All spacing is a multiple of 4px. |
| `spacing.scale` | Array: `name` (space-1 to space-16), `px`, `rem`, `tailwindClass`, `$description`. |
| `spacing.layout.contentWidth` | `1024px` — use `max-w-5xl` on page wrappers. |
| `spacing.layout.pagePaddingX` | `16px` — use `px-4` on page wrappers. |
| `spacing.layout.breakpoints` | sm=640, md=768, lg=1024, xl=1280. |
| `spacing.layout.borderRadius` | sm=4px (`rounded`), md=8px (`rounded-lg`), lg=12px (`rounded-xl`). |

Rules:
- Do not use arbitrary pixel values outside the spacing scale.
- All page-level wrappers must use `max-w-5xl mx-auto px-4`.

### 6.5 Motion tokens (`motion.*`)

| Path | Content |
|------|---------|
| `motion.duration.fast` | 150ms — hover states, focus rings. |
| `motion.duration.base` | 300ms — default transitions. |
| `motion.duration.slow` | 500ms — page-level transitions. |
| `motion.duration.logoFrame` | 10 000ms — animated logo frame hold. Do not modify. |
| `motion.easing.standard` | `cubic-bezier(0.4, 0, 0.2, 1)` |
| `motion.easing.decelerate` | `cubic-bezier(0, 0, 0.2, 1)` — entering elements. |
| `motion.easing.accelerate` | `cubic-bezier(0.4, 0, 1, 1)` — exiting elements. |

Rules:
- All animations **must** have a `prefers-reduced-motion` alternative.
- Animate `transform` and `opacity` only; never animate layout properties.
- Animated logo must not autoplay when `prefers-reduced-motion: reduce` is set.

### 6.6 Logo tokens (`logos.*`)

| Path | Content |
|------|---------|
| `logos.primaryLogo` | `"square/square"` — the canonical logo. |
| `logos.usageRules` | Clear space, minimum sizes, format selection, theme selection, prohibitions array. |
| `logos.square.variants` | Square logo variants with id, skinTone, animated, asset objects. |
| `logos.horizontal.variants` | Horizontal variants with id, withText, theme, animated, asset objects. |

Rules:
- Clear space: min 25% of logo width on all sides.
- Minimum digital width: 80px (square), 200px (horizontal).
- Dark background → use a `-dark` theme variant.
- Never modify logo colors, proportions, or apply opacity.

### 6.7 Component catalog (`design-guide/components.jsonld`)

Each entry has `@type: ComponentSpec`, `elmModule`, `props[]`, `tokenDeps[]`.

When generating UI code:
1. Fetch `design-guide/components.jsonld` — if a component exists, **import it** rather than writing inline HTML.
2. Use the `props` array to construct the correct Elm record.
3. Do not override a component's token-driven colors or spacing with inline styles.
